# github_finder_flutter

GitHub リポジトリを検索するための Flutter アプリです。

※このリポジトリは、 [株式会社ゆめみ Flutter エンジニアコードチェック課題](https://github.com/yumemi-inc/flutter-engineer-codecheck?tab=readme-ov-file) に対する回答として作成されました。

## 要件

- Flutter SDK（最新の安定版）
- Dart SDK（最新の安定版）

## セットアップ

1. リポジトリをクローン
   ```bash
   git clone https://github.com/kanyamo/github-finder-flutter.git
   ```

2. プロジェクトのディレクトリに移動
   ```bash
   cd github_finder_flutter
   ```

3. 依存パッケージをインストール
   ```bash
    flutter pub get
    ```

4. pre-commit フックを有効化
    ```bash
    pre-commit install
    ```

5. アプリを実行
    ```bash
    flutter run
    ```

## 機能

本アプリは、要件で定義された動作はもちろん、それ以上の利便性を提供するために以下の機能を実装しています。

### リポジトリ検索

- リポジトリの検索を行うことができます。検索方法には、基本検索機能と詳細検索機能があります。
- 検索可能なリポジトリは、GitHub の公開リポジトリのみです。

#### 基本検索機能

- 基本検索機能では、キーワードを用いて検索が行えるようになっています。
- キーワードは、リポジトリ名、説明、topicsの中から検索されます。

#### 詳細検索機能

- 基本検索画面の右上、フィルターアイコンをタップすると、**詳細検索画面**に遷移します。
- 詳細検索機能では、キーワードに加え、リポジトリのユーザー名、organizationでの検索が行うことができます。
- さらに、リポジトリ名や説明、topics, READMEの中それぞれについてキーワード検索を行うか否かを個別に指定できます。
- 詳細検索画面を閉じても検索条件は保持されますが、詳細検索を行った後に基本検索を行った場合、キーワードのみが検索条件として使用されます。その後再度詳細検索画面を開くと、前回の詳細検索条件が復元されます。

### 検索結果の表示

- 検索結果にリポジトリ名とオーナーのアイコン、Star数が一覧表示されます。
- リポジトリ名をタップすると、リポジトリの詳細情報を表示する画面に遷移します。

### リポジトリの詳細情報の表示

- リポジトリの詳細情報（名前、オーナーアイコン、説明、言語、Star数、Watcher数、Fork数、Issue数）を表示します。
- Issue数はオープン中のもののみを表示します。
- "このユーザーが作成したリポジトリを検索する" ボタンで、オーナーのリポジトリを検索することができます。
- "View on GitHub" ボタンで、GitHub のリポジトリページを開くことができます。
    - コードの閲覧や Issue の作成など、GitHub の機能を利用することができます。

### 設定

設定画面から、

- 言語(英語・日本語)
- テーマ(ライト・ダーク)

を変更することができます。
変更は即時反映され、デバイスに保存されるのでアプリを再起動しても設定は保持されます。

## 状態管理パッケージについて

このアプリでは、[Riverpod](https://pub.dev/packages/riverpod) を使用して状態管理を行っています。

## 開発におけるAIサービスの利用について

このアプリの開発において、[Claude 3.5 Sonnet](https://claude.ai/)と[ChatGPT 4o](https://chatgpt.com/)を使用しました。

- Claude 3.5 Sonnet
    - 採用理由
        - 精度が高く、Artifactsという機能でコードが扱えるため
        - コンテキストが長いため。つまり、破綻なくアプリを開発するために必要な情報を提供できるため
    - どのように使ったか
        - アプリ開発初期のディレクトリ構造の作成・設計において、ディレクトリ構造の提案を依頼しました。
        - 基本となるコードを作成してもらい、それをたたき台にアプリの開発を進めました。
        - なるべく具体的な指示を出し、精度の向上を図りました。
    - プロンプトの例
        ```
        これは、GitHubのリポジトリ検索ができるアプリの一部です。現状、詳細検索画面はその内部で検索条件の状態管理が完結していますが、これには

        * 通常検索で使用したキーワードが引き継げない
        * 詳細検索をもう一度しようと思うと、全ての条件をもう一度設定する必要がある

        などの問題があります。これを解消するために、Riverpodの機能を使ってキーワード入力欄や詳細検索の状態などを管理するようにアップデートしてください。詳細画面を閉じても、状態が残っているようにしてください。
        ```
- ChatGPT 4o
    - 採用理由
        - 生成の質が高いため
        - OpenAIのAPIキーを所有しており、Cursorエディタとの連携が可能だったため
    - どのように使ったか
        - コンポーネントに対する部分的な改修・機能追加などで、Cursorエディタ上でコードを生成してもらいました。
